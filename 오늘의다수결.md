# 오늘의 다수결 — 앱인토스 미니앱 기획서

> 단계: **다작 3번** · 예상 MVP: **3주** · 심사 리스크: **매우 낮음(풀 전체 최안전군)** · 데일리 훅 현실성: 3/5
>
> 한 줄 컨셉: **매일 자정 공개되는 일상 밸런스 질문 1개('부먹 vs 찍먹')에 3초 투표하고, 전국·연령대별 결과를 다음 날 확인하는 취향 통계 콘텐츠.**

## 1. 왜 이 앱인가

- 1:1 상호작용·매칭·사행성·UGC가 전혀 없는 **순수 집계형** — 심사 통과가 가장 확실한 카테고리.
- 투표/집계형 후보 6개(밸런스 한판, 몇대몇, 눈치왕 등) 중 심사관 평가에서 **훅 설계가 가장 정교**하다고 판정된 안: "어제 결과 + 오늘 질문" 동시 갱신의 이중 훅 + "즉시 보기 = 광고" 게이트.
- 보상형 광고 게이트 전환율과 **데일리 콘텐츠 운영 부담**(매일 질문 1개)을 실측하는 실험체 — 운세 팩트체크도 매일 콘텐츠를 발행해야 하므로 직결되는 학습.

## 2. 코어 루프 (데일리 훅)

- 매일 0시: **새 질문 공개 + 어제 질문 결과 발표가 동시에** 일어남 → 투표하러 들어왔다가 어제 결과까지 확인하고 나감(이중 훅).
- 투표는 3초(양자택일 탭 한 번). 결과는 **다음 날 공개가 원칙** → 재방문이 구조적으로 강제됨.
- **즉시 보기 = 보상형 광고** ("기다리거나, 광고 보거나" — 카카오페이지 기다무 공식).
- 결과 화면: 전국 비율 + "당신은 27% 소수파" + 연령대·성별 분포(광고 게이트 뒤).
- 푸시: 정오 1회 "어제는 73%가 찍먹 — 오늘의 질문 도착" (미투표자 회수).
- 7일 연속 투표 스트릭 + 월간 "내 취향 리포트"(다수파 적중률, 소수파 횟수).
- 적용된 리텐션 공식: ③시간 리셋(결과 대기), ④매일 갱신 콘텐츠, ②스트릭.

## 3. MVP 기능 범위

**포함**
- 일일 질문 1개 투표 + 익일 결과 공개 + 즉시 보기 광고 게이트
- 전국/연령대/성별 분포 (인구통계는 토스 로그인 동의 기반 — 돛단배 profiles 패턴)
- 연속 투표 스트릭, 지난 질문 아카이브(읽기 전용)
- "나는 소수파" 결과 카드 공유
- 운영자 질문 큐 관리(질문 100개 선작성 후 출시)

**제외(2차)**
- 유저 질문 제안(채택 시 출제자 표기) — 받을 경우 금칙어 필터+신고 모더레이션 필요
- 브랜드 협찬 질문(식품·OTT 광고주 연계)
- 친구와 답 비교

## 4. 토스 플랫폼 연동

- **공유 리워드**: "난 소수파 27%였다" 카드 공유 → 다음 결과 즉시 보기 1회권. 소수파 판정 직후가 공유 욕구 최고점.
- **토스포인트 프로모션**: 7일 연속 투표 스트릭 달성 시 소액 1회 (서버 중복 지급 방지).
- **스마트발송**: 정오 리마인드 1회/일. 미투표 세그먼트 타겟팅.
- **광고 게이트**: 결과 즉시 보기 + 연령대·성별 상세 분포 — 돛단배 `useAdGate` 그대로 재사용.

## 5. 수익화

- **보상형 광고**: 결과 즉시 보기, 상세 분포 열람 (핵심 인벤토리 — 매일 1회 이상 자연 발생)
- 결과 화면 **전면/배너**
- (2차) 브랜드 협찬 질문 — 트래픽 확보 후

## 6. 심사 주의사항

- **토스 행운퀴즈와 외형 구분 소명**: 지식 정답·포인트 퀴즈가 아니라 "정답 없는 취향 통계"임을 UI와 검수 설명에 명확히. "퀴즈" 워딩 자체를 피하고 "투표/다수결"로 통일.
- **질문 운영 가이드 필수**: 정치·연애(만남)·종교·민감 주제 절대 배제. 음식·생활습관·소비·콘텐츠 취향 위주.
- 1:1 대화·매칭 요소 전무 → 소개팅 리스크 0.

## 7. 기술 설계 메모

- 스택: Granite + React 18 + TDS Mobile + Supabase.
- 스키마(단순): `questions`(공개일 인덱스), `votes`(user_id, question_id unique — RLS로 본인 행만 insert), 집계는 결과 공개 시점 머티리얼라이즈(매 조회 count 금지).
- 일일 전환(KST 자정)은 돛단배 KST 카운터 패턴. 투표 마감/공개는 서버 시간 기준.
- 인구통계: 토스 로그인 동의 → `toss-auth` Edge Function이 profiles 기록 (돛단배 구조 그대로 — 클라 위변조 가드 포함).
- 재사용: `useAdGate`, `share.ts`, `toss-auth`, `analytics.ts`, KST SQL 패턴. 유저 출제 도입 시 금칙어·신고 모더레이션도 재사용.

## 8. 리스크와 대응

| 리스크 | 대응 |
|---|---|
| 질문 큐 고갈(운영 부담) | 출시 전 100개 선작성 + 주 단위 배치 작성. 2차에 유저 제안으로 파이프라인화 |
| 초기 유저 적어 "전국 통계" 재미 반감 | 표본 N 미만이면 비율 대신 "현재 투표 중" 연출, 운영자 시드 투표 |
